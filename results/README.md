# Results Directory

This directory stores simulation output generated by the MAPD toy model.

Simulation runs are automatically saved into timestamped subdirectories containing collected Mesa `DataCollector` outputs such as:

- model-level statistics
- agent-level statistics
- collision metrics
- task completion data

Example structure:

```text
results/
├── README.md
├── 20260527_193055/
│   ├── model_data.csv
│   └── agent_data.csv
├── 20260527_201422/
│   ├── model_data.csv
│   └── agent_data.csv
```

## Git Tracking

Generated CSV files are intentionally excluded from version control to avoid committing large volumes of simulation data.

Only this README file and the directory structure should be tracked by Git.

Recommended `.gitignore` entry:

```gitignore
results/*
!results/README.md
```

## Purpose

The collected data will later be used for:
- benchmarking MAPF/MAPD algorithms
- analysing collision frequency
- evaluating congestion behaviour
- comparing planning strategies
- measuring task throughput
- visualising simulation performance over time
