# Results Directory

This directory stores simulation outputs generated by the MAPD toy model.

Each simulation run is automatically saved into a timestamped subdirectory containing collected Mesa `DataCollector` outputs and simulation log files.

Generated outputs may include:

- model-level statistics
- agent-level statistics
- collision metrics
- task completion data
- reservation/debugging logs

Example structure:

```text
results/
├── README.md
├── 20260527_193055/
│   ├── model_data.csv
│   ├── agent_data.csv
│   └── simulation.log
├── 20260527_201422/
│   ├── model_data.csv
│   ├── agent_data.csv
│   └── simulation.log
```

## Logged Information

Simulation log files currently include:

- task generation
- token interactions
- task assignment
- reservation updates
- reservation conflicts
- worker movement
- path planning
- collision events
- worker state transitions

The logging system is primarily intended for debugging and analysing multi-agent coordination behaviour, particularly reservation conflicts and collision scenarios.


## Git Tracking

Generated simulation outputs are intentionally excluded from version control to avoid committing large volumes of generated data.

Only this README file and the directory structure should be tracked by Git.

Recommended `.gitignore` entry:

```gitignore
results/*
!results/README.md
```

## Purpose

The collected data and logs will later be used for:

- benchmarking MAPF/MAPD algorithms
- analysing collision frequency
- evaluating congestion behaviour
- debugging reservation-table behaviour
- comparing planning strategies
- measuring task throughput
- analysing token coordination behaviour
- visualising simulation performance over time
- evaluating future Token Passing implementations
